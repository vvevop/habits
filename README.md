# Трекер привычек

Это простое веб-приложение для отслеживания привычек. Оно состоит из бэкенда на FastAPI и фронтенда на чистом JavaScript, с упором на правильную конфигурацию и легкое развертывание.

## Особенности

- Создание, отметка о выполнении и удаление привычек.
- Простой интерфейс с аутентификацией по имени пользователя (хранится локально).
- REST API для управления привычками.
- Бэкенд на FastAPI с базой данных SQLite.
- Фронтенд на ванильном JS, HTML и CSS.
- Гибкая конфигурация через переменные окружения (`.env`) и файлы-примеры.

## Структура конфигурации

Проект использует файлы-примеры для конфигурации. Вы можете скопировать их и настроить под свои нужды. Локальные файлы конфигурации добавлены в `.gitignore`, чтобы избежать попадания секретов в репозиторий.

- `.env.example` -> `.env` (переменные окружения для бэкенда)
- `config.py.example` -> `config.py` (логика чтения конфигурации)
- `config.js.example` -> `config.js` (конфигурация фронтенда)
- `nginx.conf.example` -> `nginx.conf` (конфигурация Nginx)
- `habits.service.example` -> `habits.service` (конфигурация сервиса systemd)

## Установка и запуск

### 1. Клонирование репозитория
```bash
git clone https://github.com/vvevop/habits.git
cd habits
```

### 2. Настройка бэкенда
1.  **Создайте файлы конфигурации:**
    Скопируйте файлы `.env.example` и `config.py.example`:
    ```bash
    cp .env.example .env
    cp config.py.example config.py
    ```
    Откройте файл `.env` и при необходимости измените `APP_HOST`, `APP_PORT` или `CORS_ORIGINS` для вашей локальной среды.

2.  **Создайте и активируйте виртуальное окружение:**
    ```bash
    python -m venv .venv
    # Для Linux/macOS
    source .venv/bin/activate
    # Для Windows
    .venv\Scripts\activate
    ```

3.  **Установите зависимости:**
    ```bash
    pip install -r requirements.txt
    ```

4.  **Запустите сервер:**
    ```bash
    uvicorn main:app --reload
    ```
    Сервер будет доступен по адресу, указанному в `.env` (по умолчанию `http://127.0.0.1:8000`). Документация API (Swagger UI) находится по адресу `http://127.0.0.1:8000/docs`.

### 3. Настройка фронтенда
1.  **Создайте файл конфигурации:**
    Скопируйте `config.js.example`:
    ```bash
    cp config.js.example config.js
    ```
2.  **Настройте URL API:**
    Откройте `config.js` и раскомментируйте строку с `API_URL` для локальной разработки, чтобы она указывала на ваш запущенный бэкенд:
    ```javascript
    // Было (для продакшена):
    // API_URL: 'https://api.your-production-domain.com/api/habits'

    // Стало (для локальной разработки):
    API_URL: 'http://localhost:8000/api/habits',
    ```

3.  **Откройте `index.html`:**
    Вы можете открыть `index.html` прямо в браузере. Для избежания проблем с CORS убедитесь, что в файле `.env` в переменной `CORS_ORIGINS` указан `null` (для файлового протокола) или адрес вашего локального сервера (например, `http://localhost:3000`).

## API Эндпоинты

- `GET /api/habits?username={username}`: Получить все привычки.
- `POST /api/habits`: Создать привычку.
- `PUT /api/habits/{habit_id}`: Обновить статус привычки.
- `DELETE /api/habits/{habit_id}?username={username}`: Удалить привычку.

## Развертывание

Для развертывания используйте и настраивайте файлы `nginx.conf` и `habits.service` на вашем сервере, создав их из соответствующих `.example` файлов.
